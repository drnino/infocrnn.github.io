Todos los archivos una vez se completó el entrenamiento del modelo de la red Neural fueron subidos a un nuevo repository en github y posteriormente se realiza el deployment a un servidor online optimizado para archivos en phyton.

La serialización y deserialización del modelo se realiza mediante la librería pickle con la finalidad de mantener un archivo comprimido del modelo entrenado.

Una vez deserializado el modelo para su uso recibe la información proveniente del cuestionario que se expresa en la página web. En este paso el objetivo es obtener datos en forma de una matriz cuyas dimensiones correspondan al input del modelo. Luego se utiliza un hash table para contrastar la información de la matriz unidimensional obtenida del formulario con el resultado del modelo preentrenado y así mantener el menor tiempo posible en la obtención de las respuestas. Posteriormente el resultado se carga nuevamente en la página inicial.

Para convertir la información obtenida del formulario inicial se utiliza una aplicación en flask que posteriormente es la que se utilizará como input para el modelo de red neural.

Para reducir la complejidad de la utilización de la red neural se creó una página web única con un formulario que solo tiene valores preestablecidos de sí y no, lo que evita las posibles complicaciones asociadas a la libertad del usuario de intriducir valores aleatorios. Y posteriormente el resultado de la predicción de la red neural se presenta en la misma página de forma sencilla.

Una vez el modelo alcanza un área bajo la curva mayor o igual a 99.6% utilizando la base de datos inicial ( de entrenamiento y prueba) el modelo preeliminar es expuesto a una nueva base de datos hasta el momento desconocida y es evaluada nuevamente su sensibilidad y especificidad. Si los resultados no son satisfactorios el modelo preeliminar es nuevamente preentrenado recalibrando el peso de cada input utilizando el algoritmo que creamos para reentrenamiento no supervisado hasta lograr los resultados esperados.Luego el modelo es exportado y subido al servidor para su uso.

Se realiza un proceso de limpieza de datos en la base de datos inicial, ésta es dividida en un 70% de datos para aprendizaje y un 30% para prueba. Luego se procede a realizar el aprendizaje a través del modelo de red convulocional recurrente y utilizando el algoritmo de reajuste de pesos que creamos para así obtener un modelo con un área bajo la curva mayor o igual a 99.6%.
